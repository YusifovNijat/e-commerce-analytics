# E-commerce Analytics

## Project Overview

This project demonstrates a complete end-to-end analytics workflow using a 1M+ transaction e-commerce dataset.

The objective was to transform raw transactional data into structured business insights through:
- PostgreSQL — Data modeling & aggregation
- Python (Pandas, Matplotlib) — Growth analysis & trend modeling
- Power BI — Executive dashboard & KPI storytelling

The project simulates a real-world analytics scenario where stakeholders want to understand:

- Revenue trends over time
- Monthly growth volatility
- Performance stability
- Average order value behavior
- Best and worst performing months

## Dashboard Overview

Executive Dashboard Includes
- KPI Cards
- Total Revenue
- Total Transactions
- Global Average Order Value
- Best Growth Month (dynamic)
- Worst Growth Month (dynamic)
- Revenue vs Growth % (dual-axis chart)
- Year slicer for dynamic filtering


Allows business stakeholders to:
- Identify growth spikes
- Detect revenue declines
- Analyze seasonality
- Monitor financial stability
- Evaluate performance by year

Key Business Insights

- Revenue remains relatively stable with moderate volatility.
- Several months demonstrate >10% growth spikes.
- Revenue declines are typically followed by recovery periods.
- Average Order Value remains consistent (~1.4K), indicating pricing stability.
- Growth volatility is higher than transaction volatility.


This project demonstrates:
- End-to-end analytical thinking
- SQL-based data modeling
- Python-driven trend enhancement
- KPI automation
- Executive-level dashboard storytelling
- It reflects a real-world analytics workflow delivering insights to leadership.
